Paperkit alpha 0.5 Paperkit alpha 0.5 Quick start Components Demos Tutorial Discussion Spec with demos Paperkit is a Material Design web framework that works on top of native and standard technologies: HTML, CSS and JavaScript

Our main goal is to provide a simple and fast way to create webpages and apps following the latest guidelines from Google. We do not use any other framework or CSS preprocessor (like jQuery or Sass).

Important: include before every other css or js that overwrites Materializer styles and scripts
<head>
    <link rel="stylesheet" href="<path>/materializer/materializer.css">
    <script src="<path>/materializer/materializer.js"></script>
</head>
Note: <path> is the folder containing the Materializer folder. If you installed it with bower, the path will be 'bower_components'. Also, remember that the Materializer folder has '-min' at the end when it is the minified version. 3- Load and initialize Paperkit
Note: you can choose a different variable name other than 'paperkit'
var paperkit = new Materializer(); // Instantiate Paperkit on a variable of your choice
window.addEventListener('load', function(){ paperkit.init() }); // Initialize Paperkit on window load
 Next step: tags and attributes Paperkit works with its own tags and attributes.
They are precceded by the "md-" prefix.
The structure looks like this: <md-tag md-attribute="value"></md-tag>

For example: 
<md-toolbar md-color="red" md-fill="white">
	<md-row>
		<md-icon-button md-image="icon:menu"></md-icon-button>
		<md-text>I'm a toolbar!</md-text>
		<md-space></md-space>
		<md-icon-button md-image="icon:favorite"></md-icon-button>
	</md-row>
</md-toolbar>
Here we use a few Paperkit custom tags: <md-toolbar>, <md-row>, <md-icon-button>, <md-text> and <md-space>.
We can also find some Paperkit attributes: md-color, md-fill and md-image.

Most of the tags and attributes can be used anywhere in the DOM, but there are some that can only be used under certain conditions.
For example, you can only use the md-image attribute on image-related tags like <md-icon> or <md-icon-button>.
Also, <md-row> is only used on some elements including <md-toolbar> and <md-form>. Back Next step: basic layout Considerations
In order to avoid problems with some browsers and resolutions (specially in mobile webpages and apps), we have the following recommendations: Top level elements
Childs of body tag, they define the basic layout.
They are <md-toolbar>, <md-sidemenu>, <md-fab> and <md-content>. 
<body>
	<md-toolbar></md-toolbar>
	<md-sidemenu></md-sidemenu>
	<md-fab></md-fab>
	<md-content></md-content>
</body>
These elements are available under the Paperkit namespace (remember that you can change it when loading the framework with var paperkit = new Paperkit()). The only difference is that they drop the 'md-' prefix.
For example: paperkit.toolbar, paperkit.sidemenu, paperkit.fab and paperkit.content.

IMPORTANT: These elements should be unique, and included in the page when the framework inits. They should not be deleted or moved (in the DOM, not by style). You can still use them freely having these limitations in consideration (for example, you can change color, content or attributes, and use its JavaScript methods).
This doesn't mean that you must include all of them, but if you plan to use one of these elements only after the framework inits, just load it hidden from the start, and show it when you need it. If you won't use it at all, you don't have to include it.

As an exception to this rule, you can use a toolbar inside the content tag. The paperkit.toolbar namespace won't point to that toolbar, it will contain the top-level one instead. Content elements
Childs of top-level tags, mainly the md-content tag.
They can be any element, from <md-list> to <md-input>. 
<body>
	<md-toolbar>
		<md-row>
			(...)
		</md-row>
	</md-toolbar>
	<md-sidemenu>
		<md-list>
			(...)
		</md-list>
	</md-sidemenu>
	<md-fab></md-fab>
	<md-content>
		<md-list>
			<md-tile>
				<md-icon md-image="icon:android" md-fill="green"></md-icon>
				<md-text>I'm a list tile on the content tag!</md-text>
			</md-tile>
		</md-list>
	</md-content>
</body>
 Back Next step: scripting Elements initialization
The Paperkit elements on the DOM must be initialized. This proccess is automatic when the elements are already on the DOM when Paperkit inits (if you didn't customize the way of initializing Paperkit, it should happen on window load).

When you add new Paperkit custom elements (for example, loading pages with AJAX), you must init them by using paperkit.initElement(element). This function inits the element and all the elements in it. When an element has already been initialized, it won't be initialized again.

For example, let's say you have retrieved a page by AJAX and you store it in a variable named 'ajaxResponse' (note: it is a string). Then you insert it on <md-content>. This means you'll have to initialize the <md-content> tag. 
// Paperkit.content contains the <md-content> tag. We fill it with ajaxResponse
paperkit.content.innerHTML = ajaxResponse;
// After adding the HTML, we must initialize the content tag and all its elements
paperkit.initElement(paperkit.content)
Note that <md-content> tag has already been initialized (if it wasn't, it wouldn't be available at paperkit.content). This means that the element won't be initialized again, but this does not stop its children from being initialized.
