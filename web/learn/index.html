 Basics #
Paperkit provides a set of tags, attributes and JavaScript APIs.
The prefix is "md" (Material Design): <md-tag>, md-attribute="value" and md.api.

For example:
HTML 
<md-toolbar md-color="red">
	<md-row>
		<md-icon-button md-image="icon:menu"></md-icon-button>
		<md-text id="toolbar-title">Screen type: desktop</md-text>
		<md-space></md-space>
		<md-icon-button md-image="icon:favorite"></md-icon-button>
	</md-row>
</md-toolbar>
JavaScript 
window.addEventListener('md-device-changed', function(){
	var device = md.viewport.device();
	var title = document.getElementById('toolbar-title');
	title.textContent = 'Screen type: ' + device;
});
Live result Screen type: desktop

<meta charset="utf-8">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="mobile-web-app-capable" content="yes">

Check also this guide on mobifying from html5rocks. Component initialization #
Some Paperkit components must be initialized. This proccess is automatic when they are already present on window load. However, when you add new HTML with Paperkit components (for example, by loading pages with AJAX), the initialization is manual.
Use md.init() for this purpose:

The parameter is the element or array of elements to be initialized. The function runs throughout the whole subtree, meaning all children will be processed. The input element does not need to be a Paperkit component, it can be a standard <div> containing components.

For example, you retrieve a page by AJAX or create some HTML, and you store it in the 'newHTML' variable. Then you inject it into a <div> and initialize it. 
// We fill the div element with newHTML
div.innerHTML = newHTML;
// After adding the HTML, we must initialize the div element
md.init(div)
Note: components won't be initialized twice, but this won't stop their children from being initialized. Any questions? Wanna tell us something?
