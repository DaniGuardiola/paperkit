Note: this dev guide is made for a linux OS, but you should be able to develop on other platforms.
Paperkit is built using HTML, CSS and JavaScript, but the architecture is a little more complex. Follow this guide to set up the neccesary development enviroment:

1 - Install Git, NPM and Grunt 
sudo apt-get install git npm
sudo npm install -g grunt-cli
2 - Clone Paperkit: 
git clone https://gitlab.datatrends.es/opensource/materializer.git
3 - Go to Paperkit folder and update npm packages 
cd Paperkit
npm install
4 - Run Grunt from inside the Paperkit folder 
grunt

If everything works, you should see a bin/ folder, and three folders inside: paperkit/, paperkit-min/ and paperkit-files/ Project structure
Note: the structure may have changed since the last time this guide was updated.
Paperkit folder has the following structure:

source/ - the code lives here
- attribute/ - attribute JSON-to-CSS files (more info later)
- css/ - CSS files with generic styles
- import/ - generic JSON files with re-usable data for JSON-to-CSS (more info later)
- javascript - JavaScript files including the general namespace and component classes
- resources - font files, cursor images and extra icons
- tag/ - component JSON files (that compiles to CSS)
- tools/ - a few tools like JSON generators (example: a script that extracts the colors from the official Material Design spec and outputs the colors.json import)

bin/ - the compiled framework files, after grunt is executed
- paperkit/ - Paperkit folder ready to use
- paperkit-min/ - same but minified
- paperkit-files - compiled files without concatenation

labs - a playground to test and develop Paperkit components
tasks/ - contains our custom grunt tasks, currently only the JSON to CSS compiler
web/ - the webpage files (this folder will probably move to another repository in a future)
license/ - licensing stuff
grunt-materializer/ - MDCSS grunt plugin, obsolete and discontinued, we will refactor it in the future
node_modules/ - this is the NPM packages folder JSON to CSS? Whaaaat?
Paperkit code only has CSS and JS files in the ready to use package, but for a simpler development workflow we don't code CSS directly (except for general styles inside source/css/ folder).
We made our own grunt plugin that converts JSON files to CSS. The JSON format for Paperkit is an abstraction of CSS that lets us create a structure closer to our components design without worring much about CSS selectors.

For example, consider a simplified version (for example's sake) of md-color attribute, with only a "red" value. This will look like this: 
<div md-color="red"></div>
To select any element containing the attribute "md-color" with value "red", we use this CSS selector: 
[md-color="red"] {
    background-color: red;
}
Instead of doing this, we write this piece of JSON on the file source/attribute/md-color.json: 
{
  "type": "attribute",
  "name": "md-color",
  "values": [
    {
      "name": "red",
      "css": [
        {
          "property": "background-color",
          "value": "red"
        }
      ]
    }
  ]
}
After running grunt, we can find the file bin/paperkit-files/attribute/md-color.css that contains the CSS code that's right before the JSON example.

It may seem complicated but it actually makes development (and life) far easier when you understand how it works. Read the following specification to get a deep understanding of the JSON structure. JSON-to-CSS specification

Styling
CSS
Array that contains objects with a property name and its value. Always optional. 
"css": [
	{
		"property": "<property>", // Example: "background-color"
		"value": "<value>" // Example: "#fff"
	}
]
Responsive
Array that contains objects with a device target and a CSS array. Always optional. 
"responsive": [
	{
		"target": [ <targets> ], // Check source/settings.json to see all available targets. Example: [ "mobile", "tablet" ]
		"css": <CSS array>, // Optional
		"fixes": <Fixes array> // Optional
	}
]

Selecting
The selector where styling applies varies depending on the context where they are located in the JSON. The following objects create contexts.
Attribute
Can be top level or inside an attributes array of a tag. Selects attribute values. 
{
	"name": <name>, // Example: "md-color"
	"type": "attribute",
	"imports": [ // Optional. Uses a JSON file from source/import/
		{
			"name": <import name>, // Example: "colors"
			"property1": <property name>, // Example: "background-color"
			"property2": <property name> // Optional. Example: "color"
		}
	],
	"values": [ // Optional if there are imports. Array of values for the attribute
		{ 	// Value without variants
			"name": <name>, // Example: "black"
			"css": <CSS array>, // Optional
			"responsive": <Responsive array>, // Optional
			"fixes": <Fixes array> // Optional
		},
		{ 	// Value with variants. Values with variants look like this: "value-variant". Example: "red-100"
			"name": <name>, // Example: "red"
			"defaultVariant": <variant>, // Example: "500"
			"variants": [
				{
					"name": <name>, // Example: "100"
					"css": <CSS array>, // Optional
					"responsive": <Responsive array>, // Optional
					"fixes": <Fixes array> // Optional
				}
			]
			"css": <CSS array>, // Optional
			"responsive": <Responsive array>, // Optional
			"fixes": <Fixes array> // Optional
		}
	]
}
Tag
Top level object that contains the tag definition. 
{
  "type": "tag",
  "names": [ <names> ], // Can be one or more tag names, and can use a selector. Example: [ "md-element", "div[md-element]" ]
  "defaults": [ // Optional. Defines the default attributes and values
  	{
  		"name": "<property>", // Example: "md-color"
  		"values":  <values>, // Example: [ "red" ]
  	}
  ],
  "attributes": <Attributes array>, // Tag-specific attributes. The array contains regular attribute objects that will only work with this tag.
  "css": <CSS array>, // Optional
  "responsive": <Responsive array>, // Optional
  "fixes": <Fixes array> // Optional
}
Fixes
A fix provides a way to extend the selector. Use the properties "before" and/or "after" to add anything to the current selector. 
"fixes" : [
{
	"before": <selector>, // Optional. Example: "body>"
	"after": <selector>, // Optional. Example: ":not(.off)"
	"css": <CSS array>, // Optional
	"responsive": <Responsive array> // Optional
}
Using the examples above inside the "md-toolbar" tag JSON would create the following selector: 
body>md-toolbar:not(.off)
